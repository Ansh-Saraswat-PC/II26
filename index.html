System Architecture Overview

A pure single-page application built on React 18 & Vite. Strict zero-backend constraints mean all data normalization and inference happen directly in the user's browser.

Distributed Data Ingestion

  • OpenAQ v3 API: Primary source for hourly averages and live station data within the NCR bounding box.
  • WAQI Fallback: Supplements coverage gaps in sparse areas.
  • Haversine Snapping: Associates API station readings to the nearest of 272 ward centroids within a 5km radius.

Zero-Latency ML Inference

  • Algorithm: Scikit-learn DecisionTreeClassifier trained on synthetic pollution signatures.
  • Browser Deployment: Exported to ONNX via sklearn2onnx and loaded via onnxruntime-web.
  • Performance: Executes via WASM backend, achieving target sub-50ms latency per ward.

Gemini 3.0 Flash Advisory

  • Stateful Connection: The only external stateful call in the architecture.
  • Structured JSON: Uses responseMimeType to guarantee actionable, parseable outputs.
  • Streaming SSE: Ensures immediate UI feedback for policy recommendations.
